# China-market Skills 潜在问题分析

测试日期：2026-02-15  
分析范围：56个China-market skills  
分析方法：基于methodology文档、data-queries配置、实际运行测试

---

## 问题分类说明

每个skill的潜在问题分为以下类别：
1. **数据问题**：数据源不稳定、字段缺失、数据延迟等
2. **逻辑问题**：阈值设置不合理、规则冲突、边界条件处理不当等
3. **性能问题**：计算复杂度高、响应慢、资源占用大等
4. **可用性问题**：输出不清晰、缺少降级策略、错误处理不完善等
5. **A股特性问题**：未充分考虑T+1、涨跌停、停牌等A股特殊性

---

## 详细问题分析

### 1. ab-ah-premium-monitor (A/H溢价监控)

**测试结果**：✅ 通过 (1/1 views)

**潜在问题**：
1. **数据问题**：汇率数据依赖单一源（AKShare），汇率波动时可能导致溢价率失真
2. **逻辑问题**：50%极端溢价阈值可能过于保守，部分金融股长期溢价>60%
3. **A股特性问题**：未考虑沪港通/深港通每日额度限制对套利的影响
4. **可用性问题**：缺少对"一方休市另一方交易"情况的明确处理策略
5. **性能问题**：需要同时获取A股、H股、汇率三类数据，数据同步可能有延迟

### 2. block-deal-monitor (大宗交易监控)

**测试结果**：✅ 通过 (1/1 views)

**潜在问题**：
1. **数据问题**：买卖方性质数据可能不完整（部分营业部信息缺失）
2. **逻辑问题**：折价率阈值（2-5%建仓，>7%抛压）缺少历史验证
3. **可用性问题**：methodology文档过于简化，缺少完整的信号规则和阈值
4. **A股特性问题**：未明确说明大宗交易T+1锁定期的影响
5. **性能问题**：连续性分析需要多日数据聚合，计算复杂度较高

### 3. bse-selection-analyzer (北交所精选)

**测试结果**：✅ 通过 (4/4 views)

**潜在问题**：
1. **数据问题**：北交所数据源相对较少，AKShare覆盖可能不完整
2. **逻辑问题**：流动性过滤的"最小日均成交额"未给出具体数值
3. **可用性问题**：methodology文档过于简化，缺少具体的筛选规则和阈值
4. **A股特性问题**：北交所±30%涨跌幅限制与主板不同，未明确说明
5. **性能问题**：需要同时分析财务、流动性、估值多个维度，计算量大

### 4. concept-board-analyzer (概念板块分析器)

**测试结果**：✅ 通过 (5/5 views)

**潜在问题**：
1. **数据问题**：概念板块定义不统一（同花顺 vs 东方财富），可能导致结果差异
2. **逻辑问题**：5%相对强度阈值在震荡市可能过于敏感，产生过多信号
3. **A股特性问题**：板块内多只股票涨停时，板块指数可能失真（停牌股不计入）
4. **可用性问题**：300+概念板块数量过多，缺少自动筛选活跃板块的机制
5. **性能问题**：需要实时监控所有板块，数据量大，可能有延迟

### 5. convertible-bond-scanner (可转债扫描器)

**测试结果**：✅ 通过 (2/2 views)

**潜在问题**：
1. **数据问题**：纯债价值计算依赖票面利率和到期时间，部分转债条款复杂（如附加期权）
2. **逻辑问题**：双低指标<120的阈值在牛市可能筛选出的标的过少
3. **A股特性问题**：转债T+0交易与正股T+1的差异可能导致套利失败
4. **可用性问题**：强赎概率、下修概率的计算方法未明确（仅给出影响因素）
5. **性能问题**：需要同时分析转债、正股、财务数据，数据量大


### 6. disclosure-notice-monitor (公告披露监控)

**测试结果**：❌ 失败 (3/4 views，notice_daily_dashboard失败)

**潜在问题**：
1. **数据问题**：❌ stock_notice_report接口返回数据缺少'代码'字段，导致notice_daily_dashboard失败
2. **逻辑问题**：公告分类依赖关键词匹配，可能遗漏新型公告类型
3. **A股特性问题**：公告披露时间不固定（盘中/盘后），实时性难以保证
4. **可用性问题**：缺少对公告重要性的量化评分机制
5. **性能问题**：每日公告数量大（>1000条），全量分析耗时长
